# 🚀 项目启动指南

本文档介绍如何在本机运行和查看这个个人作品集网站。

---

## 📋 前提条件

在开始之前，请确保你的电脑已安装：

- **Node.js** (版本 18.0.0 或更高)
  - 下载地址：https://nodejs.org/
  - 验证安装：打开终端输入 `node -v`，应显示版本号

- **Git** (可选，用于克隆代码)
  - 下载地址：https://git-scm.com/
  - 验证安装：终端输入 `git --version`

---

## 🎯 方法一：从 GitHub 克隆项目（推荐）

### 步骤 1：克隆代码

打开终端（Windows 用户可以使用 CMD 或 PowerShell），执行：

```bash
git clone https://github.com/LeiQAQ/my-portfolio.git
```

### 步骤 2：进入项目目录

```bash
cd my-portfolio
```

### 步骤 3：安装依赖

```bash
npm install
```

等待安装完成，这可能需要 1-3 分钟。

### 步骤 4：启动开发服务器

```bash
npm run dev
```

启动成功后，你会看到类似以下的输出：

```
> my-portfolio@1.0.0 dev
> next dev

  ▲ Next.js 14.2.5
  - Local:        http://localhost:3000
  - Environments: .env.local

 ✓ Ready in 2.1s
```

### 步骤 5：在浏览器中打开

打开浏览器，访问：**http://localhost:3000**

🎉 现在你应该能看到网站了！

## 💻 方法二：如果你已经有项目文件夹

如果你已经从其他方式（如邮件、U盘）获得了项目文件夹：

### 步骤 1：打开终端并进入项目目录

```bash

# macOS/Linux 示例

```

### 步骤 2：安装依赖并启动

```bash
npm install

```

然后访问：**http://localhost:3000**

## ❓ 常见问题

### 问题 1：提示 "'npm' 不是内部或外部命令"

**原因**：Node.js 未安装或环境变量未配置

**解决**：
1. 访问 https://nodejs.org/ 下载并安装 Node.js
2. 安装完成后重启终端
3. 再次尝试 `npm install`

### 问题 2：安装依赖时卡住或报错

**解决**：
```bash
# 清除缓存后重试
npm cache clean --force
npm install
```

### 问题 3：端口 3000 被占用

**错误信息**：`Port 3000 is already in use`

**解决**：
```bash
# 使用其他端口
npm run dev -- -p 3001
```

然后访问：**http://localhost:3001**

### 问题 4：代码修改后页面没有更新

**解决**：
- 确保你看到的是开发服务器地址（localhost:3000），不是直接打开 HTML 文件
- 尝试强制刷新浏览器：`Ctrl + F5`（Windows）或 `Cmd + Shift + R`（Mac）

### 问题 5：GitHub 代码不是最新的

**解决**：
```bash
# 拉取最新代码
git pull origin main

# 重新安装依赖（如果 package.json 有变化）
npm install

# 重启开发服务器
npm run dev
```

## 🎨 功能特性

成功运行后，你将看到：

- ✅ **Hero 区域**：带粒子动画的炫酷首页
- ✅ **关于页面**：个人介绍、技能进度条、数据统计
- ✅ **经历时间线**：职业和教育历程展示
- ✅ **学习空间**：6个学习领域的资源链接
- ✅ **联系表单**：带验证的联系表单
- ✅ **主题切换**：点击导航栏的太阳/月亮图标切换亮暗主题
- ✅ **响应式设计**：适配手机、平板、桌面端
